# USAGE — IP Assignment (สัญญาโอนสิทธิ์ในทรัพย์สินทางปัญญา)

## ใช้เมื่อไร

ใช้เมื่อ SoloCorp ต้องการให้พนักงาน ผู้รับจ้าง freelance/contractor หรือ contributor โอนสิทธิ์ในทรัพย์สินทางปัญญาที่สร้างขึ้นระหว่างทำงานให้ SoloCorp

กรณีสำคัญ:
- พนักงานประจำ — ใช้ตอน onboarding
- Freelance/Contractor — ใช้เป็นส่วนหนึ่งของสัญญาจ้าง
- Open Source Contributor — ใช้ตอนรับ contribution (ถ้าต้องการโอนสิทธิ์)
- ผู้ร่วมพัฒนา (Co-developer) — ใช้กรณี collaboration

## ขั้นตอนการใช้งาน

1. กรอกข้อมูลคู่สัญญา (ชื่อ, ที่อยู่, เลขบัตรฯ)
2. ปรับข้อ 1.1 ให้ครอบคลุมประเภท IP ที่เกี่ยวข้องกับบริการของ SoloCorp
3. **สำคัญที่สุด**: ผู้โอนต้องกรอกภาคผนวก A (ผลงานก่อนหน้า) ให้ครบถ้วน — ถ้าไม่กรอก จะถือว่าไม่มีผลงานก่อนหน้า
4. ผู้โอนต้องกรอกภาคผนวก B (โอเพนซอร์สที่ใช้) — ถ้าสร้าง software
5. เลือกข้อ 7 ให้ตรงกับประเภทความสัมพันธ์ (employee vs contractor vs contributor)
6. ให้ external counsel review (`requires_external_counsel: true`)

## ใครอนุมัติ

- legal-tulya (เจ้าของ template)
- CFO (meetoo): review กรณี contractor มีค่าตอบแทน > 300,000 บาท
- CEO (turbo): review กรณีผู้โอนเป็นผู้ร่วมก่อตั้งหรือมีส่วนสำคัญ

## ข้อควรระวัง

- **ข้อสำคัญที่สุด**: ภาคผนวก A (ผลงานก่อนหน้า) — ถ้าไม่กรอก จะมีปัญหาเรื่อง ownership ของงานที่ผู้โอนนำมาใช้
- **ซอฟต์แวร์โอเพนซอร์ส**: ต้องแจ้งและตรวจสอบว่า license ไม่ขัดกับการ commercialize (GPL, AGPL, SSPL ต้องระวัง)
- ถ้าผู้โอนเป็นพนักงานประจำตาม พ.ร.บ. ลิขสิทธิ์ มาตรา 9(2): ผลงานที่สร้างในฐานะพนักงานตกเป็นของนายจ้างโดยอัตโนมัติ แต่ **สิทธิบัตร** ตาม พ.ร.บ. สิทธิบัตร มาตรา 11 ต้องมีสัญญาเป็นหนังสือ — สัญญานี้จึงจำเป็น
- ใช้ภาษาง่ายและชัดเจนในภาคผนวก A และ B — ไม่ใช่ภาษากฎหมาย
- กรณีผู้รับจ้าง (contractor): ข้อ 7.1 เวอร์ชัน contractor — ต้องระบุค่าจ้างอ้างอิงในสัญญาจ้างด้วย

## อ้างอิง

- พ.ร.บ. ลิขสิทธิ์ พ.ศ. 2537 มาตรา 9(2) (งานสร้างสรรค์ในฐานะพนักงาน)
- พ.ร.บ. สิทธิบัตร พ.ศ. 2522 มาตรา 11 (สิทธิบัตรที่ลูกจ้างทำขึ้น—ต้องมีสัญญา)
- พ.ร.บ. ความลับทางการค้า พ.ศ. 2545
- ประมวลกฎหมายแพ่งและพาณิชย์
- US template reference: Cooley IP Assignment Agreement
