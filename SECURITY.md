# Security Policy

## Reporting a Vulnerability

We take security seriously. If you discover a vulnerability in SoloCorp OS:

- **Report via GitHub Issues** — Use the `security` label
- **Alternative:** Contact Dr.SoloDev directly through GitHub

We aim to acknowledge reports within 48 hours and provide a timeline for resolution.

## Scope

- All files under `central_bus/`, `govctl_cli/`, and dependency manifests
- Agent profile configurations in `profiles/` and `.opencode/agents/`
- Pipeline orchestration scripts and service endpoints

## Out of Scope

- Third-party dependencies should be reported to their respective maintainers
- General questions about agent behavior should use standard issues

## Preferred Languages

Thai or English — both accepted.
